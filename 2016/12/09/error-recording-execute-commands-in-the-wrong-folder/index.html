报错纪录：在错误的文件夹执行命令

错误时间：2016年12月9日

错误内容：

在执行GIT命令：
             git config --global user.name "你的名字"
             git config --global user.email 你的email@XXXX.XXXX
             git init
             时没有在正确的目录下进行，而是在用户根目录下进行。

错误危害：

1、创建了冗余的文件夹；
2、有可能损坏系统文件；
3、有可能导致某些app后续无法使用。

错误教训：

在terminal中输入指令要明确当前所处的位置，不确定则弄明确了以后再进行操作，弄不明确的暂时不操作。
